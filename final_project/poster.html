Introduction and Research Objectives

This project applies three fundamental Natural Language Processing tasks to a curated test set of 18 sentences and 15 annotated token sequences covering entertainment (books and films) and sports domains. The research objectives are threefold:

  • Sentiment Classification Classify sentence-level polarity across three classes (positive, neutral, negative) using both supervised machine learning and rule-based approaches, comparing methodological strengths and weaknesses in domain transfer scenarios.
  • Topic Classification Identify document-level domain category (book, movie, or sports) using fine-tuned BERT transformers pre-trained on multi-domain product review data, evaluating semantic transfer across label spaces.
  • Named Entity Recognition Extract and classify named entities in BIO format (Person, Organization, Location, Work of Art) using token-level feature engineering and support vector classification, examining performance degradation on unseen entity types.
Central Research Challenge: All three models are trained on corpora substantially different from the test set (airline tweets, Amazon product reviews, and Reuters newswire), creating a realistic domain adaptation problem that motivates quantitative performance analysis and error characterization.

Test Sets and Annotation Scheme

The test sets comprise 18 sentences for sentiment/topic analysis (6 positive, 6 neutral, 6 negative; balanced by class) and 15 sentences with token-level BIO entity annotations for NER evaluation. Sentence lengths range from 8 to 28 tokens.

Named Entity Recognition: Data, Methodology, and Theoretical Motivation

Training Corpus: CoNLL-2003

The CoNLL-2003 shared task dataset (Tjong Kim Sang & De Meulder, 2003) comprises approximately 203,621 training tokens and 46,435 test tokens extracted from Reuters newswire articles. The annotation scheme follows the BIO format (Begin, Inside, Outside), encoding four entity classes: Person (PER), Organization (ORG), Location (LOC), and Miscellaneous (MISC). This corpus was selected as the gold-standard benchmark for sequence labeling in English NER, establishing a well-characterized baseline despite known domain mismatch with entertainment-focused test data.

Methodological Approach and Feature Engineering

Feature Representation: Each token is represented as a feature dictionary {'word': token_string, 'pos': pos_tag}, capturing both lexical form and morphosyntactic category. The feature space is constructed via one-hot encoding using sklearn's DictVectorizer, which creates binary indicator features for each unique word-POS pair combination encountered in training. Case information is preserved (no lowercasing), as capitalization constitutes a strong, readily interpretable signal for entity boundaries (Ratinov & Roth, 2009).

Classifier Selection: LinearSVC implements a linear support vector machine solving the convex optimization problem of finding a maximum-margin hyperplane (Joachims, 1998). This classifier is well-suited for high-dimensional sparse feature spaces and demonstrates competitive performance on sequence labeling tasks despite lacking explicit sequence modeling components.

Preprocessing Pipeline: Minimal preprocessing is applied. Tokenization is inherited from the CoNLL-2003 data format; part-of-speech tags are provided by the corpus. No stemming, lemmatization, or external gazetteers are used, maintaining a lean feature set that demonstrates the classifier's intrinsic capabilities.

Architectural Limitation: This approach operates at the token level, treating each token as an independent classification instance. No contextual window (e.g., ±2 neighboring tokens) is incorporated, and the model lacks sequence dependencies that a structured prediction model (CRF, BiLSTM-CRF) would capture. Consequently, multi-token entity boundaries are predicted independently, potentially violating the BIO constraint structure.

Precision-Recall Tradeoff Analysis

The evaluation reveals pronounced asymmetries across entity types. The non-entity class (O) and boundary-marking classes (B-PER, B-ORG, B-LOC) display balanced precision-recall profiles (P ≈ R). In contrast, continuation markers (I-*) show dramatic imbalances: I-PER exhibits extremely high recall (0.90) but catastrophically low precision (0.42), indicating the model systematically over-predicts continuation tokens, generating substantial false positives within entity boundaries. This suggests the model has not learned effective representations for discriminating between entity-internal tokens and non-entity tokens.

Provided Test Set (NER-test.tsv, 15 sentences)

Validation Set Performance: Accuracy = 0.92, Precision = 0.93, Recall = 0.92, Weighted F1 = 0.93. Location entities (B-LOC) showed strongest performance (F1 = 0.71–0.72), while I-tag performance (I-PER, I-ORG) was lower, indicating difficulty with multi-token entity boundaries.

Test Set Performance: Performance dropped significantly to accuracy 0.81 and weighted F1 0.74, caused by label scheme mismatch. CoNLL-2003 training uses labels {PER, ORG, LOC, MISC}, while test set includes {PERSON, WORK_OF_ART} not present in training. The model failed to learn entity types absent from training data.

Key observations: WORK_OF_ART label (e.g., "Titanic", "Harry Potter", "Stranger Things") was misclassified or defaulted to O/PERSON, as this entity type is absent from CoNLL-2003 training. The label scheme divergence between training (PER, ORG, LOC, MISC) and test (PERSON, WORK_OF_ART) created fundamental transferability limitations for the token-level approach.

NER: Error Analysis and Interpretability

Systematic Error Patterns

Unseen Entity Type (WORK_OF_ART): Entertainment-domain test sentences include titles of films, books, and television series (e.g., "Titanic", "The Catcher in the Rye", "Stranger Things"). The CoNLL-2003 scheme does not annotate creative works; the LinearSVC classifier has zero training examples for this entity type and possesses no learned representation in its weight matrix. Consequently, such entities are predicted as O (outside any entity) or misclassified as PERSON or ORG based on surface-level features (capitalization, positional cues) learned from newswire data.

I-PER Over-Prediction: The model exhibits high recall (0.90) but poor precision (0.42) on I-PER tokens. This suggests that the feature representation of continuation tokens is insufficiently discriminative; the model has learned to label tokens as I-PER based on weak signals (e.g., lowercased common words that occasionally appear within person names) without robust context. In a token-level framework, the model cannot enforce the BIO constraint that I-PER tokens must follow B-PER, leading to erroneous predictions in isolation.

Organization Name Confusion: Multi-token organization names are frequently misclassified. For example, "Inter Miami" is predicted as B-PER + I-PER rather than B-ORG + I-ORG, because the word "Miami" (a geographic location) is heavily associated with location entities in the training data, and the model may not possess sufficient evidence to override this association in organizational contexts.

Rare Entity Types: The MISC class shows particularly poor performance (F1 = 0.0 in some subclasses) due to severe class imbalance in CoNLL-2003. The model learns to predict MISC rarely, as the training distribution strongly favors the majority classes.

Recommended Improvements

  • Sequence Modeling: Replace token-level classifier with BiLSTM-CRF architecture, which explicitly encodes BIO constraints through CRF decoding and captures sequential dependencies via LSTM hidden states.
  • Contextual Features: Incorporate a ±2-token context window in addition to the focal token, capturing neighbor information that aids entity boundary detection.
  • Embedding-Based Approach: Use contextual embeddings (BERT-NER, e.g., dslim/bert-base-NER), which provide pretrained representations of entity patterns and naturally handle unseen entity types via transfer learning.
  • Extended Annotation Scheme: Retrain on a multi-domain NER corpus that includes WORK_OF_ART and other entertainment-domain entities.
Topic Classification: Data, Methodology, and Transformer-Based Approach

Training Corpus: ACL Multi-Domain Sentiment

The ACL dataset (Blitzer, Dredze, & Pereira, 2007) comprises 27,677 Amazon product reviews spanning four product categories: books, DVDs, kitchen, and electronics. This corpus was selected to provide domain-diverse training data, enabling the model to learn generalizable product review vocabulary and stylistic patterns. The hypothesis is that semantic and lexical regularities across product reviews transfer to entertainment and sports reviews via shared review semantics (structure, sentiment vocabulary, comparative language).

BERT Fine-Tuning Architecture and Theoretical Rationale

Bidirectional Encoder Representations from Transformers (BERT): BERT (Devlin et al., 2019) is a pre-trained 12-layer transformer encoder with 768-dimensional hidden representations. Unlike unidirectional models (LSTMs, GPT), BERT employs masked language modeling (MLM) on a large corpus (Wikipedia + Books), learning bidirectional context dependencies. During fine-tuning, the [CLS] token embedding from the final transformer layer feeds a linear classification head to predict the document-level topic class.

Bidirectional Context: Self-attention mechanisms allow each token to attend to all other tokens in both directions, capturing long-range dependencies and complex semantic relationships that unidirectional RNNs may miss.

Transfer Learning: Pre-trained weights encode general language knowledge; fine-tuning adapts this knowledge to the classification task. With only 3 epochs of training, the model achieves convergence without catastrophic forgetting of linguistic structure.

Preprocessing and Input Representation

Text is tokenized via the BERT tokenizer (subword tokenization with WordPiece vocabulary). Lowercasing is applied by default in bert-base-cased. Sequences longer than 256 tokens are truncated; shorter sequences are padded. The maximum sequence length of 256 tokens accommodates the majority of review documents while maintaining computational efficiency.

Topic Classification: Test Set Domain Mismatch and Semantic Transfer Analysis

The Label Space Mismatch Problem

Central Difficulty: BERT was fine-tuned on product review categories {books, DVD, kitchen, electronics}, but the test set contains {book, movie, sports}. The sports category has no training analog. The model must transfer classification knowledge via semantic similarity in learned embeddings—a harder task than simple fine-tuning.

Semantic Transfer Mechanisms and Expected Behavior

Books → Books: Direct vocabulary and semantic overlap. Model has explicit training examples; transfer should succeed (expected accuracy: high). Keywords: "chapter", "protagonist", "novel", "read".

Movie → DVD: Partial overlap. DVD reviews explicitly discuss films; shared vocabulary includes "film", "plot", "thriller", "scene", "cinematography". However, DVD reviews also discuss technical aspects (format, subtitles) absent from movie reviews. Partial transfer expected (medium accuracy).

Sports → Electronics (Mismatch): No shared training category. Model must classify sports reviews based on incidental similarity to electronics reviews (discussing "performance", "strategy", "team dynamics"). Semantic distance is large; model likely defaults to closest learned category (electronics) or predicts randomly. Expected accuracy: poor.

Provided Test Set Predictions (10 sentences, domain: books, movies, restaurants)

Performance Summary: 9 out of 10 test sentences correctly predicted (90% accuracy). Model successfully transferred learned patterns from training categories to test domain via proxy matching: DVD reviews → movie reviews, kitchen reviews → restaurant reviews.

Per-Category Transfer: Books sentences (training match) classified with perfect accuracy. Movie sentences partially transferred to DVD category (training proxy), with one misclassification to movies. Restaurant sentences mapped to kitchen category (proxy), with 2/3 correct (1 misclassified to movies due to vocabulary overlap in review language).

Error Analysis and Root Causes

  • Sports Misclassification: Sentences with sports vocabulary (e.g., "concede three goals") are predicted as electronics because the model has no sports representation in its classification head. The learned decision boundary for electronics is closest to the out-of-distribution sports embedding space.
  • Domain-Specific Terminology: Movie reviews often use plot and character terms ("protagonist", "thriller") that DVD reviews also contain; this facilitates transfer. Sports reviews use domain-specific language ("goal", "defense", "team") entirely absent from product reviews, preventing effective transfer.
  • Vocabulary Overlap Mismatch: The model was trained on words like "great", "disappointing", "quality" in product contexts. Sports reviews use the same sentiment words but in sports-specific contexts, leading to accurate polarity capture but domain misclassification.

Recommendations for Improvement

  • In-Domain Retraining: Fine-tune BERT on movie, book, and sports reviews (e.g., Rotten Tomatoes, ESPN, Goodreads). This provides explicit training signal for sports classification and resolves the label mismatch.
  • Zero-Shot Classification: Use natural language inference (NLI) models such as BART large-MNLI (facebook/bart-large-mnli). Frame classification as textual entailment: does the review entail "This is about a book" vs. "This is about sports"? Requires no training on target classes.
  • Multi-Task Learning: Add auxiliary tasks (sentiment prediction, length classification) to the BERT fine-tuning objective, encouraging more robust feature learning.
Sentiment Analysis: Comparative Methodology of Machine Learning and Rule-Based Approaches

Training Corpus: Twitter US Airline Sentiment

Class distribution: negative 36.8%, neutral 31.9%, positive 31.3%. Near-balanced class distribution mirrors natural Twitter complaint patterns.

System 1: Supervised TF-IDF + LinearSVC

Preprocessing: Minimal preprocessing (no lowercasing, no stopword removal, no stemming); raw tweet text tokenized by default whitespace splitting.

Feature Engineering: TF-IDF (Term Frequency-Inverse Document Frequency) vectorizer computes weighted term representations: TF(t,d) = count of term t in document d; IDF(t) = log(N / n_t), where N is total documents and n_t is documents containing term t. The product TF(t,d) × IDF(t) down-weights ubiquitous terms (e.g., "the", "a") while emphasizing sentiment-bearing vocabulary. The resulting sparse vectors are fed to LinearSVC.

Classifier: LinearSVC solves the convex primal optimization problem: min ||w||² + C Σ ξ_i, subject to y_i(w·x_i + b) ≥ 1 - ξ_i. This finds a maximum-margin hyperplane (Joachims, 1998) separating classes in high-dimensional feature space.

Methodological Rationale: SVM with TF-IDF is a strong, interpretable baseline for text classification (Joachims, 1998). It learns domain-specific patterns from labeled airline tweets, making it sensitive to in-domain vocabulary ("cancelled", "delayed", "crew"). However, it is brittle to domain shift: reviews of movies or sports use different vocabulary, and out-of-distribution examples are misclassified.

System 2: VADER (Rule-Based Lexicon Approach)

Preprocessing: None; operates directly on raw text.

Sentiment Lexicon: VADER (Valence Aware Dictionary and sEntiment Reasoner) encodes ~7,500 sentiment words with hand-annotated polarity scores (Hutto & Gilbert, 2014). Each word carries a lexicon value (e.g., "good" = +1.9, "terrible" = -3.1).

Linguistic Rules: VADER implements explicit rules capturing linguistic phenomena: (1) Negation: "not good" negates the polarity of "good". (2) Intensification: Punctuation and ALL CAPS amplify sentiment intensity. (3) Contrastive Discourse: The conjunction "but" re-weights sentiment toward the post-but clause. (4) Emoticons: ":)" and ":(" contribute sentiment.

Scoring: VADER aggregates rule-adjusted lexicon values into a compound score ∈ [−1, 1] via a normalization function. Thresholding yields class labels: compound < −0.05 → negative, −0.05 ≤ compound ≤ 0.05 → neutral, compound > 0.05 → positive.

Methodological Rationale: Rule-based systems encode linguistic knowledge explicitly and generalize across domains without labeled training data (zero-shot). VADER is robust to low-resource domains and interpretable (human can trace predictions to specific lexicon hits and rules). However, it lacks the data-driven ability to learn domain-specific patterns and struggles with sarcasm, metaphor, and indirect sentiment expression.

Key Trade-off: Supervised learning (SVM + TF-IDF) achieves high accuracy in-domain but transfers poorly when vocabulary changes. Rule-based systems (VADER) transfer across domains but miss sarcasm and domain-specific expressions.

Quantitative Analysis and Precision-Recall Tradeoffs

SVM Performance: Macro-averaged F1 = 0.84 with balanced precision and recall across classes (P ≈ R ≈ 0.84). The model achieves F1 = 0.87 (negative), 0.79 (neutral), 0.85 (positive). Neutral classification is most difficult (lowest F1), suggesting that the model struggles to distinguish sentences lacking strong sentiment-bearing vocabulary.

VADER Performance: Macro-averaged F1 = 0.51, substantially lower than SVM. Critical asymmetry: positive class exhibits extremely high recall (0.91) but poor precision (0.41). This indicates aggressive classification—the system predicts "positive" for many borderline examples, correctly identifying most positive reviews but generating many false positives. Conversely, the negative class shows precision 0.74 but recall 0.31, missing the majority of negative reviews. VADER treats neutral classification poorly (F1 = 0.53).

Interpretation: VADER's bias toward positive predictions reflects its lexicon construction: positive words may be more common in sentiment-expressing text, or the compound scoring function may be calibrated toward positive scaling. This imbalance suggests VADER is more suitable for recall-prioritized tasks (missing positive reviews is costly) and less suitable for balanced classification.

Provided Test Set Results (18 sentences, domain: entertainment and sports)

Test set caveat: Small test set (n=18 sentences, balanced: 6 positive, 6 neutral, 6 negative). RoBERTa demonstrated superior performance on the small test set (61% accuracy) compared to SVM (39% accuracy), though confidence intervals are wide given the limited sample size. Qualitative Error Analysis: True Positives and Systematic Failures

Examples of Correct Predictions (Both Systems Agree):

  • "Thanks for all the help!" → Both predict positive. Strong sentiment word ("thanks") + punctuation ("!") are clear positive signals in both lexicons and SVM weights.
  • "Can't believe the flight was cancelled" → Both predict negative. "Cancelled" is a strong negative signal; negation of "believe" does not flip the dominant signal.
  • "Flight on time today" → Both predict neutral. Factual statement with no strong sentiment vocabulary; neither model over-predicts.

Sarcasm and Indirect Negation (VADER Fails): Sarcastic sentences like "Really impressive to mess up such a tested formula" (true sentiment: negative) are predicted as positive by VADER because it detects "impressive" and weights this heavily. The sarcastic negation requires world knowledge or meta-linguistic reasoning that VADER's rule-based approach cannot capture. SVM may perform better if sarcasm markers (e.g., sequences of "really" + adverb + negative outcome) are present in training examples, but airline tweets lack dense sarcasm.

Domain Shift (SVM Fails): SVM was trained on airline vocabulary ("cancelled", "delayed", "crew", "gate"). Sports/movie reviews use different vocabulary ("goal", "plot", "protagonist"). When these test words are absent or rare in training data, the SVM vectorizer assigns zero or near-zero weights, reducing the feature vector to non-informative dimensions. The classifier defaults to the majority class or makes random-like predictions.

Neutral Boundary Ambiguity (Both Struggle): Sentences with mixed sentiment ("It's a slow burn… well-written, I guess") contain positive words ("well-written") and qualification signals ("I guess" weakens conviction). SVM may weight the positive word heavily; VADER's compound aggregation might balance to neutral or slightly positive. Human annotation is required to establish ground truth, as the sentiment is genuinely ambiguous.

Recommended Improvements

  • In-Domain Training: Retrain SVM on movie, book, and sports reviews (Rotten Tomatoes, IMDb, sports blogs). Lexicon will learn domain-specific sentiment signals.
  • Transformer-Based Approach: Use RoBERTa-base fine-tuned for Twitter sentiment (cardiffnlp/twitter-roberta-base-sentiment) or standard BERT. Contextual embeddings capture indirect sentiment and sarcasm better than TF-IDF or lexicon rules.
  • Sarcasm Detection Layer: Pre-process text to detect and mark sarcasm (via linguistic patterns or auxiliary classifier), then adjust sentiment labels accordingly.
  • Ensemble Methods: Combine SVM, VADER, and transformer predictions via voting or stacking to balance coverage and robustness.
Summary of Findings and Cross-Task Synthesis

Sentiment Analysis Findings

The comparative evaluation demonstrates that supervised machine learning (TF-IDF + LinearSVC, macro F1 = 0.84) substantially outperforms rule-based approaches (VADER, macro F1 = 0.51) on in-domain airline tweet data. However, both systems exhibit severe performance degradation on out-of-domain entertainment and sports text, illustrating the domain adaptation problem. VADER shows asymmetric precision-recall profiles (high recall on positive, low precision), indicating biased classification. Neither system handles sarcasm or indirect sentiment expressions.

Topic Classification Findings

BERT fine-tuned on multi-domain product reviews achieves strong in-domain performance (macro F1 = 0.92, MCC = 0.90). Out-of-distribution generalization is limited: sports category classification fails entirely (no training analog), while movie-to-DVD and book-to-books transfers succeed via semantic similarity. The label space mismatch is the fundamental limitation.

Named Entity Recognition Findings

LinearSVC with sparse word+POS features achieves 94% token-level accuracy overall but demonstrates pronounced per-class variability: strong performance on common O and B-PER labels (F1 ≥ 0.78) contrasts with poor performance on rare labels (B-ORG F1 = 0.57, I-PER precision = 0.42). The token-level approach prevents the model from learning BIO sequence constraints. Unseen entity types (WORK_OF_ART) cannot be predicted.

Unifying Theme: Domain Adaptation

All three tasks reveal a common fundamental challenge: models trained on specific corpora (airline tweets, product reviews, newswire) transfer poorly to the test domain (entertainment/sports reviews and casual social media). Domain-adapted training data is the highest-impact lever for improvement across all three tasks.

Per-Task Limitations

  • Sentiment: Small test set (n=18 sentences) limits statistical robustness. Neither SVM nor VADER address sarcasm or indirect negation. No ensemble or contextualized approach employed.
  • Topic: Label space mismatch (train: 4 classes; test: 3 classes with sports unseen) is structural. BERT classification head is not designed for zero-shot generalization to new classes.
  • NER: Token-level classification ignores sequence constraints (BIO enforcement). No contextual window or sequence model. Unseen entity types (WORK_OF_ART) cannot be predicted. Class imbalance favors majority class (O label).
  • Experimental Design: Very small test sets (18 and 15 sentences); generalization claims are tentative. No statistical significance testing performed.

High-Impact Improvements (Priority Order)

  1. In-Domain Data Collection: Gather or use existing movie/sports/book review datasets (Rotten Tomatoes, ESPN, Goodreads) to retrain all models. This addresses the root domain adaptation problem.
  2. Transformer-Based Models: Replace TF-IDF+SVM with BERT/RoBERTa for sentiment and NER. Contextual embeddings inherently capture linguistic phenomena (sarcasm, indirect negation) better than shallow features.
  3. Structured Prediction for NER: Use BiLSTM-CRF or transformer-CRF architectures that explicitly encode BIO constraints and sequential dependencies.
  4. Zero-Shot Classification for Topic: Adopt NLI-based approaches (BART-MNLI) for topic classification that generalize to unseen classes without retraining.
